# Contributing to MindMelt
Hello! We are grateful for the time you are taking to improve our extension! All kinds of contributions are welcomed whether it is bug fixes, suggestions or simply improving documentation.

## Getting Started
1. Fork the repository into your own Github.
2. Clone your fork:
```
git clone https://github.com/ossd-s26/MindMelt-Extension.git
cd MindMelt-Extension
```
3. We recommend creating a new branch for your work to make the main project stable and reviewing changes easier:
```
git checkout -b feature/your-feature-name
```
4. Load the extension in Chrome for testing:
- Go to 'chrome://extensions/'
- Enable **Developer Mode**
- Click **Load unpacked**
- Select the project folder

## Making Changes 
- Make any changes locally.
- Reload the extension in 'chrome://extensions/' after the updates.
- Test on YouTube videos in order to make sure the effects still work properly.
Commit your changes with a clear and concise message:
```
git add. 
git commit -m "Update/Changed: description"
```
Push your branch to your fork:
```
git push origin feature/your-feature-name
```

## Opening a Pull Request
1. Go to the original **MindMelt** repository on GitHub.
2. Open a Pull Request from your branch.
3. Include:
- What changes were made.
- How it improves the extension or what it fixed.
- Screenshots or notes if needed.
4. Make sure your code is readable and follows the structure of the project.

## Best Practices
- Keep code clear.
- Use descriptive variable and function names.
- Focuse each pull request on one feature/fix.
- Update the documentation if functionality changes.
- Please give constructive feedback during reviews.

## Code of Conduct 
By contributing, you agree to help maintain a respectable and open environment. Any sort of harassment, discrimination or inappropriate behavior will not be tolerated. 
Review the project's Code of Conduct document for full details.
